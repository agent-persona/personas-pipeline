# exp-5.06 — Human Protocol Design

**Branch:** `exp-5.06-human-protocol-design`
**Guide:** Guide 2 — Synthesis Pipeline Architecture
**Date:** 2026-04-11
**Status:** **PARTIAL — INFRA BUILT, AWAITING PROLIFIC LAUNCH.** Loop is end-to-end testable from synthetic labels; remaining work is operational (Prolific account, budget, study upload, label collection).

## Goal

Produce the canonical human ground truth pipeline so that every downstream
LLM-judge metric in batches 1–4 (and beyond) can be calibrated against
inter-rater human agreement.

The unstated risk in batches 1–4: every WEAK/STRONG verdict was scored by
Claude-as-judge with no validation that Claude-as-judge tracks human raters.
Without this experiment landing, we don't know which prior findings sit
above the judge-noise floor and which are artifacts.

## What was built

### `evals/human_protocols/` package (new)

| File | Purpose |
|---|---|
| `README.md` | Operator-facing protocol descriptions, selection criteria, manual launch steps |
| `protocols.py` | Canonical `Protocol` dataclasses for all 3 candidates (A: blind matching, B: pairwise preference, C: forced-choice ID) |
| `agreement.py` | From-scratch Cohen's κ + Krippendorff's α (ordinal & nominal). Zero scipy/numpy dependency. |
| `test_agreement.py` | 5 unit tests on hand-worked inputs. **All pass.** |
| `build_stimuli.py` | Generates Prolific-ready CSVs from existing batch-4/5 transcripts |
| `ingest.py` | Reads Prolific label CSVs, computes agreement per protocol, picks the winner |
| `prolific_config.yaml` | Study-config template (audience, pay, screening, attention checks) for the operator |

### The three protocols

**Protocol A — Blind matching.** Rater sees a 6-turn transcript and picks
which of 3 candidate persona-cards drove the bot side. Cohen's κ on the
3-way nominal label.

**Protocol B — Pairwise preference.** Rater sees the same probe answered
by two twins (e.g., baseline vs treatment from exp-1.15), picks which
feels more like the described persona, with a `tie` option. Krippendorff's
α on the ordinal scale `a_better < tie < b_better`.

**Protocol C — Forced-choice persona ID.** Rater sees one twin reply plus
4 candidate persona-cards, picks which persona wrote the reply. Cohen's
κ on the 4-way nominal label.

### Stimuli generation — actually ran

`build_stimuli.py` was executed against the existing exp-4.13 conversations
and exp-1.15 baseline/treatment paired probes. Output:

| Protocol | n items | Source |
|---|---|---|
| A — blind matching | 18 | exp-4.13 conversations × 3 6-turn slices per transcript |
| B — pairwise preference | 20 | exp-1.15 baseline/treatment paired probes for the same 10 adversarial inputs |
| C — forced-choice ID | 10 | exp-1.15 personas + their probe replies |

CSV files are committed at `output/experiments/exp-5.06/stimuli_protocol_{a,b,c}.csv`.

**Sample size caveat in this run:** the mock crawler only produces 2
clusters (engineers + designers), so Protocol A and Protocol C had to pad
with synthetic distractors to reach 3 and 4 candidates respectively. A real
pilot requires expanding `crawler/crawler/connectors/ga4.py` to ≥4 distinct
mock cohorts before uploading. The CSVs include a `SYNTHETIC DISTRACTOR`
marker on every padded card so the operator cannot accidentally upload them.

### Agreement math — verified

`test_agreement.py` runs 5 cases against hand-worked inputs:

```
PASS cohen_kappa_perfect_agreement
PASS cohen_kappa_zero_agreement (k=0.000)
PASS cohen_kappa_published_example (k=0.346)
PASS krippendorff_alpha_perfect
PASS krippendorff_alpha_ordinal (alpha=0.515)
```

### Ingest pipeline — dry-run validated

`ingest.py` was dry-run on a synthetic label set generated by sampling
70%/15%/15% (true label / tie / opposite) for 20 items × 5 raters on
Protocol B. Result:

```json
{
  "winner": "b_pairwise_preference",
  "winner_agreement": 0.382
}
```

α = 0.38 is exactly the regime expected for 70%-noisy synthetic labels —
moderate but not strong agreement, well below the α ≥ 0.80 we'd want for
a canonical protocol. The dry-run files were deleted after verification;
this run only confirms the math and CSV-shape is correct.

## What is NOT done

- **No real human labels.** Prolific account creation, budget approval,
  study upload, and the 2–4 hour wait for the studies to fill have not
  been performed. Operator action required.
- **No `calibrated_judge_confidence()` in `evaluation/judges.py`.** That
  function depends on a winning protocol's agreement number, which doesn't
  exist until step (7) in the launch checklist below.
- **No re-scoring of batch-4 experiments against the calibrated judge.**
  Same dependency.
- **The mock corpus is too small** for a real pilot. As noted above, the
  crawler needs ≥4 distinct cohorts before Protocols A and C have real
  distractors.

## Manual steps to launch (operator checklist)

1. **Expand mock data**: add ≥2 more cohorts to `crawler/crawler/connectors/ga4.py`
   so segmentation produces at least 4 clusters. Re-run `build_stimuli.py`.
2. **Review CSVs** at `output/experiments/exp-5.06/stimuli_protocol_{a,b,c}.csv`
   for any sensitive content. Remove all rows containing `SYNTHETIC DISTRACTOR`.
3. **Create Prolific studies** using the fields in `prolific_config.yaml`.
   Fill in `prolific_study_id` for each.
4. **Upload each CSV** as study stimuli. Set `n_participants` per the YAML
   (default 5/study).
5. **Wait** for the studies to complete (~2–4 hours each, ~£150 total at
   the per-participant pricing in the YAML).
6. **Download labels** and place at `output/experiments/exp-5.06/labels_protocol_{a,b,c}.csv`
   with columns `item_id, rater_id, response`.
7. **Run** `python -m evals.human_protocols.ingest`. This writes
   `FINDINGS_data.json` with α/κ per protocol, picks the winner, and
   reports majority-vote labels for spot-checking.
8. **Add `calibrated_judge_confidence(protocol)` to `evaluation/judges.py`**
   returning the winner's α/κ.
9. **Re-run** every WEAK/STRONG batch-4/5 experiment's judge against the
   calibrated number. Update each FINDINGS.md with a confidence band.

## Recommendation

**This experiment is the gating dependency for batch 5+ to be statistically
trustworthy.** The infrastructure is built; the operational launch should
take priority over starting batch 6. Without the calibration number, every
"WEAK / DEFER" and "MIXED / CONDITIONAL" verdict in batch 5 has an
unbounded error bar.

Specifically, batch 5 verdicts that this experiment will most likely
move:

- **exp-1.15 (edge-case behaviors)** — currently MIXED / CONDITIONAL with
  Δ=−0.30 in aggregate but +0.20 on matched probes. The 0.30 delta is
  almost certainly within Claude-as-judge noise; if α<0.6 on the relevant
  protocol, the experiment was effectively a coin flip.
- **exp-4.13 (length matching)** — currently SOFT-ADOPT with Δ=+0.15 to
  +0.18 across modes. These deltas are right at the judge-noise threshold;
  human calibration could either confirm a real ranking or collapse it
  into noise.
- **exp-2.07 (order-of-fields)** — currently DEFER. The qualitative
  "money-quote stability" finding is independent of the judge but the
  vocabulary deltas need calibration.
