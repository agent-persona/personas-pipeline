Schema Layer

Canonical Record Types

Contracts between crawler and ingestion. Every record carries evidence_pointer, crawl_run_id, observed_at, created_at.

👤
account
Identity anchor. One per {platform, platform_user_id}. Immutable key.
📸
profile_snapshot
Point-in-time mutable fields. Append-only. Enables drift detection.
🏘️
community
Subreddit, server, channel, group. Metadata + member count snapshots.
🧵
thread
Conversation container. Post, tweet, Discord thread, stream session.
💬
message
Individual utterance. The core behavioral signal. Body + metadata + reply edges.
🔗
interaction
User→user edge. Reply, mention, quote, react. Derived from messages.
🪦
tombstone
Deletion/edit event. Body hash retained, content redacted. Compliance.
platform/ ├── community_id/ │ ├── thread_id/ │ │ ├── message records (body, score, reply_to, depth) │ │ └── interaction records (reply | mention | quote | react edges) │ └── community metadata (description, member_count, rules) ├── _users/ │ ├── account (platform_user_id, username, first_observed) │ └── profile_snapshot[] (bio, karma, communities_active — append-only) └── tombstone[] (deleted | edited | suspended — body hash, reason)

Exhaustive user ID extraction via message history, archived threads, welcome logs. Rate-limit aware.
MEDIUM — account enumeration

HIGH — Reddit library
