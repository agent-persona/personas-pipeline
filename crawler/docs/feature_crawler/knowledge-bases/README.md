# Knowledge Bases

## Purpose

This folder holds cross-feature knowledge for the whole product.

Use it for:

- methodology
- product direction
- benchmark design
- market and category framing
- privacy, trust, compliance
- shared prompts and research workflows

Do not dump feature-local implementation notes here if they belong in a `feature_*` folder.

## Routing rule

Put knowledge in the narrowest useful place.

- feature-specific learning -> that feature's `KNOWLEDGE.md` or `plans/presearch.md`
- cross-feature or directional learning -> this folder
